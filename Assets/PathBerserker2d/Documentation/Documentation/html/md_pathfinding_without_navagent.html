Finding a path without a NavAgent following it

You can create your own paths without using a NavAgents PathTo()/UpdatePath() methods. This can be helpful when you want to check if a path exists at all, or how long it would be without the NavAgent trying to follow the path.

PathRequest request = new PathRequest(agent);
NavSegmentPositionPointer startPointer;
PBWorld.TryFindClosestPointTo(start, agent, out startPointer);
NavSegmentPositionPointer goalPointer;
PBWorld.TryFindClosestPointTo(goal, agent, out goalPointer);
request.start = startPointer;
request.goals = new[] { goalPointer };
PBWorld.PathTo(request);

This does requires an instance of NavAgent. NavAgent plays a huge role in determining which links and segments are walkable. Trying to pathfind without one wouldn't make a lot of sense. You can create a "dummy" NavAgent and use it for your pathfinding requests.

The PathRequest will not be completed immediately after calling PBWorld.PathTo(). Pathfinding happens asynchrone on a different thread. Poll a path requests current state like this:

if(request.Status == PathRequest.RequestState.Pending)
{
// not ready yet. still processing
}else if(request.Status == PathRequest.RequestState.Finished)
{
// finished.
var path = request.Path;
}else if(request.Status == PathRequest.RequestState.Failed)
{
// request failed
var failReason = request.FailReason;
var closestReachablePosition = request.closestReachablePosition;
}
